Was die CPU der GPU schickt (JS)

// im Loop, jeden Frame, EINMAL pro draw call:
material.uniforms.uTime.value  = 0.00;
material.uniforms.uColorA.value // THREE.Color
  .setRGB(0.04, 0.10, 0.22);
material.uniforms.uColorB.value
  .setRGB(0.43, 0.88, 0.78);
Schreib nie uniforms.uTime = t (ohne .value): das ersetzt das uniform-Objekt und die Bindung bricht still. Genau diese Stelle ist der Hyrum-Moment des Tages.

Der Shader, den sie füttern (GLSL)

// VERTEX
varying vec2 vUv;
void main() {
  vUv = uv;                 // uv kommt von three
  gl_Position = projectionMatrix
    * modelViewMatrix
    * vec4(position, 1.0); // alles injiziert
}

// FRAGMENT
uniform float uTime;
uniform vec3  uColorA;
uniform vec3  uColorB;
varying vec2  vUv;     // interpoliert!
void main() {
  float t = vUv.y
    + sin(vUv.x * 6.0 + uTime) * 0.06;
  vec3 col = mix(uColorA, uColorB, t);
  gl_FragColor = vec4(col, 1.0);
}

Du hast oben keine dieser Zeilen geschrieben. ShaderMaterial schreibt sie vor deinen Code. RawShaderMaterial schreibt nichts davor: dann deklarierst du jede Zeile selbst. Gleiches Bild, anderer Besitzer der Komplexität. Der Diff ist die Lektion.
